## Fixture realism contract (closes a v1 gap)

v1 fixtures were toy `{"name","action","columns"}`. **Real osqueryd rows carry a 9-key envelope**
(verified against the live `results.log`):
`action, calendarTime, columns, counter, epoch, hostIdentifier, name, numerics, unixTime`. Every fixture
below uses realistic shapes:

- **Differential (launchd, admin, etc.):** include `counter` (baseline-discard depends on it), `action`
  `added`/`removed`, and the real `columns`. launchd columns are
  `{disabled, keep_alive, label, path, program, program_arguments, root_directory, run_at_load, start_interval, username, watch_paths, working_directory}`
  — note **`program` is empty**; the command is in `program_arguments`. (Live pack currently emits 11
  columns via COALESCE and **no `removed` key**; v2's query is a *replacement* adding `program_arguments`
  \+ `removed:true` — see plan Task 4, flagged as a behavioral change.)
- **Evented file_events:** outer `action` is always `"added"`; the real FSEvents verb is `columns.action`
  (`CREATED`/`UPDATED`/`DELETED`), with `columns.{category, target_path, sha256, time}`.
- **es_launchd_writes:** `columns.{dest_filename, event_type, filename, parent, path, pid, time}` — no
  `label`.
- **Protection state (filevault):** `columns.{encrypted, encryption_status, filevault_status, name}`.
- **Webhook body (the exact bytes the dispatcher signs):** built with the same
  `jq -cn '{event_type:"osquery.alert", host:$host, alert:{title:$title, detail:$detail}}'` the
  implementation uses — **never hand-typed JSON**, because HMAC and the `osquery-<sha256(body)[:32]>`
  request-id are over the literal bytes; a reformatted body silently breaks both signature and dedup.
- **Spool line:** `<unix_timestamp>\t<request_id>\t<url>\t<base64(body)>`.

Two harnesses (the v1 single-stub harness cannot test delivery — closed here): **(H1)** `lib.bash` drives
the real alerter with a 1-line `send_alert` stub (gate/render/digest tiering). **(H2)** a delivery
harness that **sources the real `executable_osquery-alert-dispatch.sh`** with `curl`/`openssl`/`base64`
shimmed on `PATH` and `OSQUERY_WEBHOOK_SECRET_FILE` pointed at a fixture (signing, spool, redaction,
localhost).
